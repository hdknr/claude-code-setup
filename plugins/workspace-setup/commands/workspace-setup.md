# ワークスペース初期セットアップ

ユーザーの作業用ワークスペースを対話的にセットアップします。以下の手順を順番に実行してください。

## 手順

### 1. ワークスペースディレクトリの作成

ユーザーに希望するワークスペース名を確認し、`~/Projects/` 配下にディレクトリを作成して `git init` を実行してください。

- デフォルト名: `my-workspace`
- ユーザーが別の名前を希望する場合はそれを使用

```bash
mkdir -p ~/Projects/{ワークスペース名}
cd ~/Projects/{ワークスペース名}
git init
```

### 2. CLAUDE.md の作成

ワークスペースのルートに `CLAUDE.md` を作成してください。以下の内容を含めます:

- リポジトリの目的（日々の作業を管理するワークスペース）
- 日本語で応答する指示
- ユーザーが追加したい指示があれば反映

### 3. GitHub プライベートリポジトリの作成と連携

**重要: 必ずプライベートリポジトリとして作成してください。**

作業用リポジトリには個人情報や機密情報を含む可能性があるため、非公開で作成します。

```bash
gh repo create {ワークスペース名} --private --source=. --push
```

ユーザーに以下を確認してください:

- リポジトリ名（デフォルトはディレクトリ名と同じ）
- プライベートで作成してよいか（推奨）

### 4. セットアップ完了の確認

セットアップが完了したら、以下を表示してください:

- 作成されたディレクトリのパス
- GitHub リポジトリの URL
- 次のステップのヒント（ドキュメント作成、ファイル整理など）
